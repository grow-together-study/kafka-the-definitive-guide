# Kafka: The Definitive Guide Study

이 레포지토리는 『카프카 핵심 가이드』 북스터디를 위한 실습, 기록,
그리고 **팀 기준 카프카 도입·설계·운영 문서**를 남기기 위한 공간입니다.

이 스터디는 단순 요약이나 발표 중심이 아니라,
**실습 → 토론 → 합의 → 산출물** 흐름을 통해
실무에서 바로 사용할 수 있는 판단 기준을 만드는 것을 목표로 합니다.

### 🎯 스터디 목표

- 『카프카 핵심 가이드』 완독
- 단순 요약이 아닌 **실무 도입·설계·운영 판단 기준 확보**
- 스터디 종료 시 **팀 기준 카프카 운영 문서 세트 완성**

### 🔑 핵심 운영 원칙

#### 실습 환경 원칙
- ZooKeeper 기반 예제는 **개념 이해용**
- 모든 실습·데모·산출물은 **KRaft 기준**

#### 학습 방식
- 실습은 **사전 수행**
- 모임에서는 개념 정리, 운영 판단 토론, 실험 결과 리뷰, 산출물 합의에 집중

### 🧑‍💻 파일 관리 규칙 (중요)

- 개인 실습 및 기록은 반드시 다음 규칙을 따릅니다.

  - 경로: `practice/weekX/{name}/`
  - 파일 예시:
    - `notes.md`
    - `experiment-xxx.md`

- 다른 사람의 폴더는 수정하지 않습니다.
- 개인 기록과 공용 산출물은 절대 섞지 않습니다.
- 주차별로 합의된 결과만 artifacts/에 반영합니다.

### 🐳 Kafka 실행 방법 (KRaft)

실습 환경 실행:
```bash
cd docker
./start.sh
```
- Bootstrap Server: localhost:9092

Kafka 데이터 초기화가 필요한 경우:
```bash
./stop.sh
rm -rf docker/data
```

### 🔄 실습 → PR → 산출물 흐름

1. 사전 실습 수행
2. 개인 폴더에 실습 기록 작성
3. PR 생성 (PR 템플릿 사용)
4. 모임에서 리뷰 및 토론
5. 합의된 내용만 산출물로 정리

> 실습 결과보다 **시도와 기록**을 더 중요하게 봅니다.
> 실패한 실험도 정상적인 학습 결과입니다.

### 📌 Pull Request 규칙

- PR은 정답 제출이 아니라 **사고 과정 공유** 목적
- 코드가 없어도 PR 가능
- 설정 선택 이유, 관찰 결과, 질문이 있으면 충분

PR 생성 시 자동으로 템플릿이 적용됩니다.

### 🧪 템플릿 문서

- 실험 기록 템플릿: `templates/experiment.md`

- 산출물 템플릿: `templates/artifact.md`

### 🏁 스터디 종료 목표

- 발표 자료 ❌
- 개인 노트 모음 ❌
- **실제 사용할 수 있는 카프카 운영 기준 문서 ⭕**

이 레포는 스터디 종료 이후에도
카프카 도입 검토, 운영 기준, 온보딩 자료로 활용하는 것을 목표로 합니다.
